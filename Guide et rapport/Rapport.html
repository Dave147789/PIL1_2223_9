Rapport de Projet - Gestion des emplois du temps



TimeMaster par Lheureux AZGBA

Introduction


Le présent rapport décrit notre projet de réalisation d'une application web de gestion des emplois du temps pour l'Institut de Formation et de Recherche en Informatique (IFRI), durant l'année académique 2022-2023. Notre objectif principal était de développer une solution efficace et conviviale pour les étudiants et les coordonnateurs de formation, en leur permettant d'éditer et de consulter facilement les emplois du temps.


Description Du Projet


Notre application web de gestion des emplois du temps repose sur une architecture client-serveur. Elle comprend les modules suivants :

  • Gestion des enseignements et de la masse horaire : Cette fonctionnalité est réservée aux coordonnateurs de formation, qui peuvent créer et organiser les emplois du temps en fonction des enseignements et des heures prévues.
  • Planification des emplois du temps : Les coordonnateurs de formation peuvent éditer les emplois du temps hebdomadaires, en spécifiant les cours, les enseignants et les salles de classe correspondantes pour chaque plage horaire.
  • Consultation des emplois du temps publiés : Les étudiants peuvent consulter les emplois du temps hebdomadaires qui ont été publiés par les coordonnateurs de formation. Ils peuvent filtrer les emplois du temps par filière, groupe ou enseignant pour une recherche plus facile.

Conception de l'Application


Nous avons choisi d'utiliser le framework Django en Python pour le développement de notre application web de gestion des emplois du temps. Django est un framework puissant et flexible qui facilite la création d'applications web robustes et évolutives. Python, quant à lui, est un langage de programmation populaire et polyvalent, qui offre de nombreuses fonctionnalités et une syntaxe simple et lisible.

Pour la gestion des données, nous avons opté pour MySQL comme système de gestion de base de données. MySQL est un système de base de données relationnelle fiable et performant, largement utilisé dans l'industrie. Nous avons conçu une structure de base de données relationnelle en utilisant les fonctionnalités offertes par MySQL, permettant ainsi de stocker et de gérer efficacement les informations relatives aux emplois du temps, aux enseignements, aux enseignants et aux salles de classe.

Afin d'organiser notre projet et de le rendre maintenable, nous avons adopté une structure modulaire. Notre projet Django est divisé en plusieurs modules, chacun ayant sa propre responsabilité. Voici un aperçu des modules principaux :

  • Modèles de données : Ce module contient les définitions des modèles de données qui décrivent la structure des informations stockées dans la base de données. Nous avons créé des modèles pour représenter les enseignements, les enseignants, les emplois du temps, les salles de classe, etc.
  • Vues : Les vues sont responsables de la logique métier de notre application. Elles traitent les requêtes HTTP, effectuent des opérations sur les modèles de données et renvoient les réponses appropriées. Nous avons créé des vues pour gérer la création, la modification, la suppression et la consultation des emplois du temps.
  • Templates : Les templates sont utilisés pour générer les pages HTML dynamiques de notre application. Nous avons créé des templates pour afficher les emplois du temps, les formulaires d'édition, les messages de confirmation, etc. Les templates utilisent le langage de templating de Django pour intégrer les données dynamiques dans les pages.
  • Fichiers statiques : Ce module contient les fichiers statiques tels que les fichiers CSS, JavaScript, les images, etc. Ces fichiers sont utilisés pour styliser et améliorer l'interface utilisateur de notre application.

Cette structure modulaire facilite la maintenance, l'extension et la collaboration sur notre application. Chaque module est responsable d'une partie spécifique de l'application, ce qui rend le code plus lisible et plus facile à gérer.

Ainsi, en utilisant Django, MySQL et une structure modulaire, nous avons pu concevoir une application web de gestion des emplois du temps qui répond aux besoins des étudiants et des coordonnateurs de formation de manière efficace et conviviale.

Déploiement et Utilisation

Pour déployer notre application, veuillez suivre les instructions détaillées ci-dessous :

  1. Cloner le dépôt Github du projet "PIL1_2223_9" sur votre machine locale : Ouvrez votre terminal ou votre interface de ligne de commande, puis exécutez la commande suivante : 
    git clone https://github.com/votre-utilisateur/PIL1_2223_9.git
    Assurez-vous d'avoir Git installé sur votre système pour pouvoir utiliser la commande de clonage.
  2. Assurez-vous d'avoir installé Python et Django sur votre système : Vérifiez que Python est installé en exécutant la commande suivante dans votre terminal : 
    python --version
    Si Python est installé, vous verrez sa version s'afficher. Pour installer Django, utilisez la commande suivante : 
    pip install django
    Cela installera Django et ses dépendances nécessaires.
  3. Créez un environnement virtuel et activez-le : Un environnement virtuel est recommandé pour isoler les dépendances du projet. Dans votre terminal, exécutez les commandes suivantes : 
    python -m venv mon_environnement
    source mon_environnement/bin/activate
    Cela créera un nouvel environnement virtuel et l'activera.
  4. Installez les dépendances du projet en exécutant la commande suivante : Accédez au répertoire du projet en utilisant la commande cd PIL1_2223_9, puis exécutez la commande suivante pour installer les dépendances : 
    pip install -r requirements.txt
    Cela installera toutes les bibliothèques et les modules nécessaires pour exécuter l'application.
  5. Configurez les informations de la base de données dans le fichier de configuration : Ouvrez le fichier de configuration settings.py dans le répertoire PIL1_2223_9/projet et recherchez la section de configuration de la base de données. Remplacez les paramètres par ceux correspondant à votre environnement de base de données (par exemple, nom d'utilisateur, mot de passe, hôte, port).
  6. Effectuez les migrations de la base de données en utilisant la commande : Dans votre terminal, exécutez la commande suivante pour appliquer les migrations de la base de données : 
    python manage.py migrate
    Cela créera les tables nécessaires dans votre base de données.
  7. Lancez le serveur de développement en utilisant la commande : Pour démarrer le serveur de développement, exécutez la commande suivante : 
    python manage.py runserver
    Le serveur sera lancé et vous pourrez accéder à l'application localement à l'adresse http://localhost:8000/.
  8. Accédez à l'application via l'URL fournie par le serveur de développement : Ouvrez votre navigateur web et accédez à l'URL http://localhost:8000/. Vous serez redirigé vers la page d'accueil de l'application.

En suivant ces étapes, vous serez en mesure de déployer et d'utiliser notre application sans aucun problème. Assurez-vous de suivre chaque étape avec attention pour garantir un déploiement réussi.

Un manuel d'utilisation détaillé est également disponible dans notre rapport de projet en HTML, qui est inclus dans le dépôt Github.
